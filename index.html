Introdução à criação de arquivos .XML

Neste documento, exploraremos como criar arquivos .XML utilizando a CU50 e discutiremos as razões pelas quais os arquivos .XML são utilizados nos setores de Engenharia Elétrica e Engenharia de Conceituação.

Padronização de opcionais e Nodelist padrão.

O principal objetivo de utilizarmos arquivos com extensão .XML é padronizar os opcionais que serão utilizados na carroceria. Além disso, definir qual pino do módulo multiplex será utilizado para a alimentação e controle do sistema assim utilizado um nodelist padrão para cada ZFES.

Opcionais Módulo Conector Pino
Farol de Milha MM1A Conector 1 Pino 1
Farol de Neblina MM1A Conector 1 Pino 2

Estrutura

A estrutura de um arquivo .XML é composta por tags, que são delimitadas por < e>, por exemplo.:

< name> MM1A < name>

Entre os simbolos < e>, temos o nome da tag, que é o identificador da tag, e entre os identificadores temos o valor da tag.

Ao utilizarmos uma tag dentro da outra, estamos reresentando um agrupamento de tags, por exemplo.:

< MUX>
....< Name>MM1A < name>

....< PIN_SET>
........< CN>1 < CN>
........< PN>1 < PN>
........< LOAD>10,50A < LOAD>
........< LOGIC>XXXXXXXX < LOGIC>
........< NAME>UD_USB < NAME>
....< PIN_SET>

< MUX />

O caso demonstrado acima, representa a estrutura de um arquivo .XML

    MUX
Representa o inicio e fim módulo multplex MM1A.
    PIN_SET
Representa o inicio e fim das informações pertencentes a um pino.
    CN
Representa o conector do pino.
    PN
Representa o pino do conector.

Informações do Pino

As informações como Load, logic e name, são informações que dirão como o pino ira se comportar.

    LOAD
Corrente que o pino ira enviar ou receber
OBS. No sistema multplex Lohr Amsys, caso o valor determinado seja ultrapassado, uma mensagem de erro sera exibida na HeadUnit ( Visio4-4.3" / HU10" ) e tag NFC quando exportada.
    LOGIC
    NAME
